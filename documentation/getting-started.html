Getting Started

Learn how to write, run, and understand OPSL scripts — from zero to a working multi-library program.

Installation

OPSL has no external dependencies. It uses only Node.js built-in modules (net, child_process, fs, path, vm). All you need is Node.js installed.

Requirements: Node.js v14 or later. No npm install needed.

Verify your setup:

node --version # Should show v14 or higher

Your First OPSL Script

1 Create a project folder

mkdir my-opsl-project cd my-opsl-project mkdir opsl-local/greeting mkdir opsl-local/greeting

2 Create an OPSL script

Create a file called test.opsl in your project folder:

IMPORT [js] greeting VARIABLE greeting.name = "World" CALL greeting.showMessage

This script does three things:

  • IMPORT — loads the greeting JavaScript library
  • VARIABLE — sets a variable name in the greeting library
  • CALL — executes the showMessage function

3 Create a TCP-enabled library

Create opsl-local/greeting/greeting.js:

const TCPClient = require('/path/to/tcp-server-v3/client'); function showMessage() { console.log("Hello from greeting!"); } (async () => { await TCPClient.init('greeting', 4001); await TCPClient.registerFunction('showMessage', showMessage); console.log("loaded and registered"); })();
Important: Replace /path/to/ with the actual absolute path to your OPSL project's tcp-server-v3/client directory. Each library needs this to communicate with the TCP server.

4 Run it

cd /path/to/OPSL (Copy 2) node runtimeTCP/main.js /path/to/my-opsl-project/test.opsl

You should see:

Starting fresh TCP server... [TCP Server] Server on 3000 TCP server started with clean state Importing library: greeting (js) [greeting] loaded and registered [greeting] Hello from greeting! TCP server stopped OPSL execution completed successfully

How to Write a TCP-Enabled Library

Every JavaScript library in OPSL follows the same pattern. Here's the anatomy:

Library Template

// 1. Require the TCP client (global npm package — no path needed!) const TCPClient = require('opsl-tcp-client'); // 2. Define your functions — NO PARAMETERS allowed function myFunction() { // Read current state from TCP server console.log("Hello!"); return "result"; } // 3. Set variables directly — no intermediate local vars // 4. Initialize — NO PORT NEEDED, OS assigns one automatically (async () => { await TCPClient.init('libraryName'); // Auto port! await TCPClient.set('myVariable', 'Hello World'); await TCPClient.registerFunction('myFunction', myFunction); console.log("loaded and registered"); })();

Key rules:

  • No port needed — TCPClient.init('name') only. The OS assigns a free port automatically via listen(0).
  • Register every function you want other libraries or OPSL scripts to call
  • Set initial variables with TCPClient.set() — no intermediate local variables needed
  • Print "loaded and registered" — the runtime waits for this exact string
  • No function parameters — functions are called with zero arguments. Read state from the TCP server with TCPClient.get().
  • Use require('opsl-tcp-client') — a globally installed npm package, no path resolution needed

Updating variables inside functions:

Critical: If a function modifies a variable, it MUST call TCPClient.set() to update the TCP server. Otherwise, other libraries won't see the change.
async function increment() { // Get the latest value from the server var current = await TCPClient.get('counter'); var newVal = String(Number(current) + 1); // Update on the server so other libs see it await TCPClient.set('counter', newVal); return newVal; }

The 4 OPSL Instructions

InstructionFormatWhat it does
IMPORTIMPORT [lang] nameLoads a library. lang = js, py, c, or opsl
VARIABLEVARIABLE lib.name = "value"Stores a variable in a library via TCP
ASSIGNASSIGN target.name = source.varCopies a variable from one library to another
CALLCALL lib.functionNameExecutes a function (via TCP for JS, nested .opsl for OPSL)

IMPORT

Loads a JavaScript library. The runtime searches for the file in two locations (in order):

  1. projectFolder/libraryName.js — project root
  2. opsl-local/libraryName/libraryName.js — local library folder
IMPORT [js] test → looks for test/test.js or opsl-local/test/test.js IMPORT [opsl] helper → looks for helper.opsl or opsl-local/helper/helper.opsl

VARIABLE

Sets a variable in a library. The value is sent over TCP and stored on the server.

VARIABLE test.name = "Alice" → stores test.name = "Alice" on TCP server VARIABLE add.x = "42" → stores add.x = "42"

ASSIGN

Copies a variable from one library to another. This requires TWO TCP round-trips: GET from source, SET to target.

ASSIGN test.result = add.output → get add.output, set test.result = that value

CALL

For JS libraries: sends a function call through TCP to the library's local server.
For OPSL libraries: finds and executes a nested .opsl file.

CALL test.sayHello → executes test.sayHello() via TCP CALL opsl.functionCall → dynamic dispatch: reads functionVar, executes lib.func

Full Example: Counting Loop

This example counts from 0 to 1000, printing each number. It uses two libraries: add (increments) and ifJS (checks the condition).

loop.opsl

IMPORT [js] ifJS IMPORT [js] add VARIABLE ifJS.trueReturn = "" VARIABLE ifJS.falseReturn = "LoopTest.loop" ASSIGN ifJS.number = add.output CALL ifJS.ifFunc ASSIGN opsl.consoleLog = ifJS.number CALL opsl.consoleLog ASSIGN add.number = ifJS.number CALL add.add ASSIGN opsl.functionVar = ifJS.output CALL opsl.functionCall

opsl-local/add/add.js

const TCPClient = require('opsl-tcp-client'); async function add() { var number = await TCPClient.get('number'); number = String(Number(number) + 1); await TCPClient.set('number', number); await TCPClient.set('output', number); return number; } (async () => { await TCPClient.init('add'); // No port! await TCPClient.registerFunction('add', add); await TCPClient.set('number', '0'); await TCPClient.set('output', '0'); console.log("add.js loaded and registered"); })();

opsl-local/ifJS/ifJS.js

const TCPClient = require('opsl-tcp-client'); async function ifFunc() { var number = await TCPClient.get('number'); var trueReturn = await TCPClient.get('trueReturn'); var falseReturn = await TCPClient.get('falseReturn'); console.log(number); var output; if (Number(number) > 1000) { output = trueReturn; } else { output = falseReturn; } await TCPClient.set('output', output); } (async () => { await TCPClient.init('ifJS'); // No port! await TCPClient.registerFunction('ifFunc', ifFunc); await TCPClient.set('output', 'LoopTest.loop'); await TCPClient.set('trueReturn', ''); await TCPClient.set('falseReturn', 'LoopTest.loop'); await TCPClient.set('number', '0'); console.log("ifJS.js loaded and registered"); })();

Running Scripts

# From the OPSL project root: node runtimeTCP/main.js /path/to/your/script.opsl # Or use the old entry point (delegates to runtimeTCP/): node runtime-tcp.js /path/to/your/script.opsl
Tip: Each run starts a fresh TCP server, killing old processes first. This ensures clean state with no leftover variables or port conflicts.
