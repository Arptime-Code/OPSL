Code Walkthrough

File-by-file explanation of every source file in the project — what it does, how it works, and why.

parser.js ~100 lines

Purpose: Converts OPSL text lines into structured JavaScript objects.

How it works: The splitBy() helper replaces delimiters with |, then splits. The parseLine() function checks which keyword a line starts with and extracts the relevant fields.

Key functions

  • parseLine(lineInput) — main entry point. Checks startsWith() for each keyword.
  • parseImport(trimmed) — splits by spaces and brackets, returns {type, lang, library}
  • parseAssign(trimmed) — splits by ' = ', '.', spaces — returns 5 fields
  • parseVariable(trimmed) — splits by ' = ', then left by '.', right by '"' — returns library, name, value
  • parseCall(trimmed) — splits by '.' and space — returns {type, library, name}
  • splitBy(text, replacements) — replaces each delimiter with '|', then splits

Design decision

No regex. Simple string replacement is clearer for beginners and sufficient for OPSL's predictable syntax.

tcp-workers.js ~150 lines

Purpose: Two jobs: (1) Maintain one persistent TCP socket for all runtime requests. (2) Spawn JS libraries as separate Node.js processes.

Key functions

  • getRuntimeSocket() — lazy initializer. Creates TCP connection once, reuses forever.
  • handleIncomingData(chunk) — buffers TCP data, splits on newlines, resolves matching requests.
  • sendRuntimeRequest(name, action, key, value) — sends formatted message, returns Promise.
  • createWorker(lang, name, file) — creates worker object; for JS, spawns process and waits for ready.
  • createJSWorker(worker) — kills old process, spawns new, sets up stdout listener, waits for ready.
  • setVariable/getVariable/executeFunction() — thin wrappers around sendRuntimeRequest.

Persistent socket details

// Global state: var runtimeSocket = null; // Created once, reused var socketBuffer = ''; // Accumulates partial TCP data var pendingRequests = {}; // Maps request IDs to resolve functions var nextRequestId = 0; // Increments for each request

runtimeTCP/main.js ~55 lines

Purpose: Entry point. Orchestrates the entire execution lifecycle.

Flow (5 steps)

  1. killOldProcesses() — frees all known ports with fuser -k
  2. serverManager.startServer() — spawns TCP server process
  3. fs.readFileSync(inputFile) — reads the .opsl script
  4. new Runtime(inputFile).processOPSLString() — executes the script
  5. cleanup.cleanup() — kills all library processes and TCP server

runtimeTCP/runtime.js ~60 lines

Purpose: The Runtime class — ties all modules together.

Key methods

  • processOPSLString(fileString) — splits by newlines, parses each line, dispatches by type
  • createWorker(name, lang) — delegates to library-loader
  • executeOPSLFunction(lib, name) — delegates to opsl-executor
  • getGlobalLibraries() — exposes shared state for cleanup

Dispatch pattern

switch (instruction.type) { case 'IMPORT': await instructionHandler.handleImport(instruction, this); break; case 'VARIABLE': await instructionHandler.handleVariable(instruction, this); break; case 'ASSIGN': await instructionHandler.handleAssign(instruction, this); break; case 'CALL': await instructionHandler.handleCall(instruction, this); break; }

runtimeTCP/server-manager.js ~55 lines

Purpose: Start, check, and stop the TCP server process.

Key functions

  • isServerRunning() — tries to connect to port 3000. Success = running.
  • startServer() — spawns node tcp-server-v3/server/index.js, waits 500ms.
  • stopServer(serverProcess) — process.kill(-pid) to kill process group.

runtimeTCP/library-loader.js ~40 lines

Purpose: Finds library files on disk and creates TCP workers.

Search order

  1. baseDir/libraryName.js — project root (checked first)
  2. baseDir/opsl-local/libraryName/libraryName.js — local folder (fallback)

Then calls tcpWorkerLib.createWorker(lang, name, filePath) to spawn the process.

runtimeTCP/instruction-handler.js ~95 lines

Purpose: Four handler functions — one for each OPSL instruction type.

Handler details

  • handleImport — checks duplicate → creates worker → stores in globalLibraries
  • handleVariable — looks up worker → calls tcpWorkerLib.setVariable()
  • handleAssign — GETs from source → SETs to target (skips OPSL libs)
  • handleCall — checks special cases (functionCall, consoleLog) → routes to OPSL or JS execution

runtimeTCP/opsl-executor.js ~55 lines

Purpose: Finds and executes nested .opsl files (when OPSL calls OPSL).

How it works

  1. Searches for library/name.opsl in root or opsl-local
  2. Reads the file as a string
  3. Creates a new Runtime instance (nested)
  4. Copies parent's importedLibraries so the nested runtime shares all libraries
  5. Calls processOPSLString() on the nested file

runtimeTCP/cleanup.js ~25 lines

Purpose: Gracefully shuts down everything at the end of execution.

Steps

  1. Iterates globalLibraries, calls tcpWorkerLib.closeWorker() for each
  2. Calls serverManager.stopServer() to kill the TCP server
  3. All kill() calls wrapped in try/catch for safety (processes may already be dead)

native-library/library.js ~50 lines

Purpose: Dynamic function dispatch — parses "library.functionName" strings and routes execution.

How it works

  1. Splits "library.functionName" on '.'
  2. Checks if the library is imported
  3. If OPSL: calls executeOPSLFunction()
  4. If JS: calls worker.executeFunction() (TCP remote call)

Used by CALL opsl.functionCall to dynamically execute whatever function is stored in the functionVar variable.
