API Reference

Complete reference for TCPClient methods, OPSL instructions, configuration variables, and file structure.

TCPClient API

Available via require('tcp-server-v3/client') in every library.

await TCPClient.init(name, port)

Initializes the library: starts a local TCP server on port, connects to the main server, and registers the library.

name
String — unique identifier for this library (e.g., "test", "greeting")
port
Number — unique port for this library's local server (e.g., 4001, 4002)
await TCPClient.init('test', 4001);
await TCPClient.set(key, value)

Stores a key-value pair on the TCP server under this library's namespace.

key
String — variable name
value
String — variable value (always stored as string)
await TCPClient.set('counter', '42'); await TCPClient.set('name', 'Alice');
await TCPClient.get(key)

Retrieves a stored value from the TCP server.

key
String — variable name
var count = await TCPClient.get('counter'); // "42" var missing = await TCPClient.get('nope'); // ""
await TCPClient.registerFunction(name, fn)

Makes a function callable by other libraries via callRemote().

name
String — function name for remote calls
fn
Function — the actual function to execute
function add(a, b) { return String(Number(a) + Number(b)); } await TCPClient.registerFunction('add', add);
await TCPClient.callRemote(target, fn, ...args)

Calls a function registered on another library.

target
String — target library name
fn
String — function name to call
...args
Any — arguments passed to the function (converted to strings)
var result = await TCPClient.callRemote('add', 'add', 5, 10); console.log(result); // "15"

OPSL Instructions

IMPORT Load Library

IMPORT [lang] libraryName

Loads a library. The runtime searches for the library file in two locations.

lang
js | py | c | opsl
libraryName
String — name used to reference this library in other instructions
IMPORT [js] test IMPORT [opsl] helper IMPORT [py] pythonLib

VARIABLE Store Value

VARIABLE library.variableName = "value"

Stores a variable in a library via TCP.

VARIABLE test.name = "Alice" VARIABLE add.counter = "0"

ASSIGN Copy Variable

ASSIGN targetLibrary.targetName = sourceLibrary.sourceName

Copies a variable from one library to another via two TCP round-trips (GET + SET).

ASSIGN test.result = add.output ASSIGN opsl.consoleLog = ifJS.number

CALL Execute

CALL library.functionName

Executes a function. For JS libraries: routes via TCP. For OPSL libraries: executes nested .opsl file.

CALL test.sayHello CALL greeting.showMessage CALL opsl.functionCall ← special: dynamic dispatch

Error Handling

Graceful degradation

  • Missing libraries: Instruction handlers check importedLibraries[name] and return early if not found
  • Missing OPSL files: executeOPSLFunction() logs "file not found" and returns
  • Already-exited processes: All kill() calls wrapped in try/catch
  • Port conflicts: fuser -k runs before each spawn, silencing errors with 2>/dev/null
  • Ready timeout: Promise.race([readyPromise, timeoutPromise]) prevents infinite hangs
  • TCP connection drops: runtimeSocket = null on error, next request triggers reconnect
