What is OPSL?

OPSL (One Programming/Scripting Language) is a framework that lets you write scripts in a custom domain-specific language (.opsl files) that orchestrate code execution across multiple programming languages — JavaScript, Python, C, and even other OPSL files. Instead of choosing one language, you use them all together.

Each library runs as its own isolated Node.js process, communicating with others through a central TCP server on port 3000. This provides process isolation, scalability, and the ability to distribute libraries across different machines.

Quick Example

Here's what an OPSL script looks like:

IMPORT [js] test IMPORT [js] greeting VARIABLE test.myVar = "hello" CALL test.sayHello CALL greeting.showMessage

And here's what a TCP-enabled JavaScript library looks like:

const TCPClient = require('tcp-server-v3/client'); function sayHello() { console.log("Hello!"); } (async () => { await TCPClient.init('test', 4001); await TCPClient.registerFunction('sayHello', sayHello); await TCPClient.set('myVar', 'initial'); console.log("loaded and registered"); })();
Performance: The system executes ~0.06ms per instruction thanks to persistent TCP sockets. A 1000-iteration loop completes in ~0.7 seconds.
